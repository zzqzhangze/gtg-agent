## 问题记录

| 编号  | 描述                                                            | 严重程度 | 根因 / 修复 | 状态 |
|-----|---------------------------------------------------------------|---------|------------|------|
| 7.2 | 无法找到session_id，无法测试                                           | 中 | 非 bug。DevTools → Application → Local Storage → `http://localhost:8000` → key `session_id` | 已确认 |
| 7.3 | 无法找到session_id，无法测试                                           | 中 | 同上 | 已确认 |
| 8.6 | 预览栏不会清空，AI回复只带了沙箱内部的文件地址，不包含下载链接。需要的文件下载到了后端服务启动的本地download目录 | 高 | **根因：** 前端下载链接指向 `/files/{session_id}/{name}`，该路由从 temp 目录读文件；但 `download_files` 节点把文件存到 `downloads/` 目录。两个不同目录，404 无链接。**修复：** `api.py` 挂载 `downloads/` 到 `/downloads`，`app.js` 下载链接改为 `/downloads/{name}` | 已修复 |
| 8.7 | AI回复不包含下载链接。                                                  | 高 | 同上 | 已修复 |
